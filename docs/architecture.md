# 吃啥饭 - 数据架构设计理念

> 摘要：本文档说明了项目的数据建模原则与前后端类型对齐策略，旨在保证实体与关系的清晰分离并提高系统可扩展性。

## 核心原则：实体与关系分离

- 数据库实体表只存储对象自身的客观属性
- 数据库关系表存储对象之间的多对多关联
- 前端 TypeScript 类型与数据库实体一一对应
- 前端 Pinia Store 存储当前用户与实体的关系状态
- 组件通过计算属性结合实体数据与关系状态，决定 UI 展示

## 好处
- 数据不冗余：实体信息全局唯一
- 变更隔离：修改实体属性不影响用户状态，反之亦然
- 扩展性强：新增点赞、评分等关系，只需新建关系表，不改实体表
